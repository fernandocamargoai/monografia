\chapter{Análise de Resultados}
\label{cap:resultados}

Para avaliar os resultados do sistema proposto, foram criados dados de teste que simulam o histórico de execuções de um acadêmico de Engenharia de Computação. Esses dados simulam como seria o dia-a-dia de tal acadêmico entre os dias 14/06/2014 e 15/08/2014. Ou seja, tal histórico incluiria registros feitos em uma parte do primeiro semestre de 2014, as férias do meio de ano e o início do segundo semestre. A utilização desses dados ao invés de dados reais se deve à impossibilidade de obtenção de dados reais, visto que a ferramenta proposta não está pronta para utilização. Mesmo que estivesse, seria necessário um bom período de tempo coletando dados de um usuário.

A Tabela \ref{tab:dadosDeTeste} mostra os dados de teste com as seguintes colunas: data, dia da semana (apenas para fins informativos), categoria, meta, início e fim da execução. Para fins de teste, um programa simples foi construído para importar esses dados e utilizar nos testes. Esse programa tem como entrada os dados de teste e a linha desejada. Ele obtém os dados da meta dessa linha, ignorando os momentos de execução já existentes e considera como se fosse uma meta agendada. Todas as linhas anteriores da tabela são consideradas como histórico de execução. Com isso, o programa executa o sistema proposto e mostra o horário em que o mesmo agendaria a notificação ao usuário. Assim, podemos comparar esse horário com o horário de início verdadeiro da execução.

Para avaliar o comportamento do algoritmo, foram sorteadas dez linhas da Tabela \ref{tab:dadosDeTeste} para servir de entrada ao programa de teste. Cada um desses resultados serão avaliados a seguir com uma explicação sobre o mesmo.

\begin{description}
 \item[Linha 02] \hfill \\
  Por ser a segunda execução da lista, o histórico de execuções passadas contém uma única execução. Portanto, como era de se esperar, o algoritmo retorna como resultado o momento de início dessa execução presente no histórico: 07:15.
 \item[Linha 07] \hfill \\
  Anterior à essa meta, existe uma única meta de mesma categoria. Por esse motivo, sua relevância é considerada maior que das outras execuções anteriores e o momento de início da mesma é retornado como resultado: 07:15.
 \item[Linha 10] \hfill \\
  Nesse ponto, uma considerável quantidade de metas de mesma categorias foram executadas. Assim, o sistema começa a retornar um resultado próximo da execução real. Para esse caso, uma notificação seria agendada para 18:23, enquanto essa linha apresenta momento de início de 19:02.
 \item[Linha 21] \hfill \\
  A meta de fazer caminhada foi realizada algumas vezes anteriormente a esse registro. O resultado apresentado pelo sistema foi de 08:10, enquanto o registro apresenta inicio às 06:59. Essa diferença pode ser explicada devido à linha 13, que possui a mesma meta, foi realizada no mesmo dia da semana e é o registro mais atual de execução de tal meta. Assim, esse registro teve peso determinante no resultado do sistema.
 \item[Linha 23] \hfill \\
  Para essa linha, o resultado apresentado pelo sistema foi de 22:10, enquanto o apresentado no registro foi de 18:20. Essa diferença considerável se deve à grande quantidade de execuções de metas de mesma categoria por volta das 22 horas no histórico. Comparando essa execução com as anteriores, pode-se notar que essa foi a realizada mais cedo.
 \item[Linha 28] \hfill \\
  Devido à maior presença de execuções de mesma categoria no meio de semana, o qual costuma acontecer mais tarde, o resultado do sistema foi mais tarde do que o apresentado pelo registro. O sistema apresentou 19:02, enquanto o registro apresentou 16:10.
 \item[Linha 46] \hfill \\
  Nessa linha, houve a resposta do sistema foi próxima à apresentada no registro. Enquanto o registro apresentou 22:10, o sistema sugeriu 22:56.
 \item[Linha 51] \hfill \\
  Assim como a linha de teste anterior, de mesma meta, o resultado do sistema foi razoavelmente próximo. Enquanto o registro apresentou 21:10, o sistema sugeriu 22:10, o horário de execução da Linha 46, que foi a execução mais recente e que acabou tendo maior peso.
 \item[Linha 59] \hfill \\
  O resultado apresentado para essa meta foi muito bom, visto que o sistema sugeriu 15:12 e o registro apresentou 15:34, que são horários bem próximos.
 \item[Linha 67] \hfill \\
  Essa linha demonstrou resultado razoável, visto que o registro apresentou 08:15 e o sistema sugeriu 09:37.
\end{description}

Através desses testes, pode-se concluir que os resultados do sistema foram aceitáveis. Em alguns casos, houve uma certa variação entre o resultado esperado e o resultado apresentado para o sistema. Mas, no geral, não houve variações tão grandes e injustificáveis. Vale lembrar que esses são dados simulados e não dados reais. Tal sistema precisaria ainda de uma validação real após um longo tempo de utilização por usuários reais.
